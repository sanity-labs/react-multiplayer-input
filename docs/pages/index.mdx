    caret jumps to the end of the input. This behavior results in a poor user experience, especially in scenarios where
    multiple users are editing simultaneously.
    This library addresses the issue by offering `MultiplayerInput` and `MultiplayerTextArea` components. These are
    drop-in replacements for standard input and textarea elements, designed to preserve the caret position even when
    `props.value` is updated.
